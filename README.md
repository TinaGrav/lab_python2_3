# Допущения
- При создании задач с помощью генератора используется файл, в котором заранее прописан список возможных задач
- Для создания задач из файла используется заранее добавленный в директорию json-файл
- Пользователь может создать вручную столько задач, сколько заранее напишет 
- Для завершения работы программы необходимо ввести число 5
- Для того, чтобы посмотреть функционал 2 лабораторной, нужно будет проверять как работает изменение одного из атрибутов последней добавленной задачи
- Пользователь не может изменить id, время создания задачи - при попытке их изменить пользователь увидит ошибку
- Пользователь может изменить payload (описание задачи, им должрна быть непустая строка); priority (приоритет - по умолчанию 3, может быть любое целое число от 1 до 5); status (статус задачи: "uncompleted", "in_progress", "done" или "cancelled", по умолчанию ставится uncompleted)
- При вводе некорректных данных для изменения одного из атрибутов задачи пользователь увидит ошибку


# Алгоритм работы

Класс Test:  __init__(self, id: int, payload: str, priority: int = 3, status: str = "uncompleted") — инициализация, по умолчанию приоритет равен 3, а статус задачи uncompleted;  __str__(self) обеспечивает красивый вывод задачи, функции id и created_at являются защищенны благодаря @property, @id.setter и @created_at.setter выводят ошибку при попытке изменить id и время создания задачи. Также в классе используются дескрипторы PriorityCheck, StatusCheck, DescriptionCheck, IsReadyCheck

Функция create_task(input_inf: dict) создает задачу класса Task из данных, использованных для создания задачи, используя только id и payload, задаваемые пользователем

Дескриптор PriorityCheck проверяет, что данные которые хочет ввести пользователь для нового priority находятся в диапазоне 1-5 и являются числом

Дескриптор StatusCheck проверяет, что данные которые хочет ввести пользователь для нового status "uncompleted", "in_progress", "done" или "cancelled"

Дескриптор DescriptionCheck  проверяет, что данные которые хочет ввести пользователь для нового payload являются строковыми и не пустыми 

Дескриптор IsReadyCheck  определяет готовность задачи к выполнению по статусу: если статус задачи не "done" и не "cancelled", то она готова к выполнению
