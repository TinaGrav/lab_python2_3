# Допущения
- При создании задач с помощью генератора используется файл, в котором заранее прописан список возможных задач
- Для создания задач из файла используется заранее добавленный в директорию json-файл
- Пользователь может создать вручную столько задач, сколько заранее напишет 
- Для завершения работы программы необходимо ввести число 6
- Для того, чтобы посмотреть функционал 3 лабораторной, нужно создать задачи с помощью любого из 3х методов создания задач, затем отфильтровать задачи по желаемому параметру.
- Задачи можно отфильтроавать по приоритету и статусу, а также по обоим этим параметрам вместе. 
- Пользователь не может изменить id, время создания задачи - при попытке их изменить пользователь увидит ошибку
- Пользователь может изменить payload (описание задачи, им должрна быть непустая строка); priority (приоритет - по умолчанию 3, может быть любое целое число от 1 до 5); status (статус задачи: "uncompleted", "in_progress", "done" или "cancelled", по умолчанию ставится uncompleted)
- При вводе некорректных данных для изменения одного из атрибутов задачи пользователь увидит ошибку


# Алгоритм работы

Класс TaskIterator - обеспечивает последовательный обход задач в очереди с сохранением позиции.
Принимает список задач tasks, сохраняет список во внутренний атрибут, устанавливает начальный индекс index = 0. Реализует протокол итерации Python: методы __iter__ и __next__. __next__ проверяет, что index не больше количества задач в очереди, возвращает нужную задачу и увеличивает index, если же index становится больше кол-ва задач, вызывается StopIteration


Класс TaskQueue - коллекция для хранения задач с поддержкой итерации и ленивой фильтрации. Создаёт пустой список self.tasks = []. add_task принимает объект класса Task, добавляет его в конец списка. __iter__ вызывает итератор задач TaskIterator, при этом каждый вызов создаёт новый итератор, следственно повторный обход очереди возможен. filter - ленивый фильтр, фильтрует задачи по выбранному аргументу: статусу или приоритету. Принимает параметры: status, priority (оба по умолчанию None), проходит по списку self.tasks в цикле и возвращает задачу через yield, если она подходит по выбранным параметрам.
